Red Teaming Sarvam Indus 105B

March 2026

These attacks were conducted solely to test the guardrails to their limits, so that the team at Sarvam (who are doing great work, by the way) can further strengthen and improve them.

01

Observations and Intuition

  • Although the model is trained on 22 languages, the model's "thoughts" appear to be only in English. Even when the user prompts in Hindi and the final output is in Hindi, the "thoughts" are in English, often including translations of the input.
  • In the model's "Thoughts," it often explicitly references its system prompt and guardrails.
  • Knowing the model's Mixture-of-Experts (MoE) architecture can help identify more targeted attack vectors. Since Sarvam Indus 105B likely shares architectural similarities with other MoE-based systems (e.g., DeepSeek), previously discovered vulnerabilities in those models may transfer.
  • At the moment, the model does not appear to have access to external tools such as web search (Update: this was added inbetween my tests) or code execution, so any computation must be performed internally through token-based reasoning.
  • The websearch could introduce new attack vectors. During testing the model outputted the phrase "sexy prime pairs" which was mentioned on the site prime-numbers.fandom.com but the model would not repeat the word when asked. This can be used to introduce trusted info via directed search terms.
